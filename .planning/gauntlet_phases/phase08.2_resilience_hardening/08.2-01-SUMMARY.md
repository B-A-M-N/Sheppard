## Changes Made

### src/research/archivist/crawler.py

**O1 - Retry loop:**
- Added `import time` at module level
- Replaced flat try/except (lines 61-81) with `for attempt in range(3)` retry loop
- `HTTPError`: retries on 5xx with linear backoff (1s, 2s); returns None immediately on 4xx
- `ConnectionError` / `Timeout`: retries with linear backoff; returns None after 3 failures
- Unexpected exceptions: no retry, return None immediately

**O5 - Extraction heuristics:**
- Per-line minimum lowered from `> 20` to `> 10` characters
- `skip_rest` now only activates when `len('\n'.join(filtered_lines)) < 500` at nav cue â€” structured pages with 500+ chars of body content are no longer truncated at first nav keyword

### src/research/archivist/loop.py

**O2 - Exception logging:**
- Fixed 3 bare `except: pass` blocks:
  1. `fill_data_gaps` search loop: `logger.error(f"[FAIL] Search query failed: {q}: {e}")`
  2. `execute_section_cycle` search loop: `logger.error(f"[FAIL] Search query failed: {q}: {e}")`
  3. `execute_section_cycle` ingestion loop: `logger.error(f"[FAIL] Ingestion failed for {url}: {e}")`
- All use `except Exception as e` (not bare except) to avoid catching SystemExit/KeyboardInterrupt
- Loop continues after each logged error

### tests/test_archivist_resilience.py

New test file with 20 tests covering all three fixes:
- 8 tests: `TestRetryClassification` â€” 5xx/4xx/ConnectionError/Timeout retry behavior
- 4 tests: `TestLoopErrorLogging` â€” logger.error called; loop continues after error
- 8 tests: `TestExtractTextHeuristics` â€” per-line threshold, skip_rest gating, .gov page extraction

**Implementation note:** Tests load archivist modules directly via `importlib.util.spec_from_file_location` to bypass `src/__init__.py` which imports chromadb/pydantic and causes import errors in test environments.
