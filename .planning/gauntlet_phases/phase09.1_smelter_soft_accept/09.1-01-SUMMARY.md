## Objective

Fix the soft acceptance bug: Ensure a source is only marked `condensed` if at least one valid atom was stored. If zero atoms extracted, mark source as `rejected`.

---

## Changes Made

**File:** `src/research/condensation/pipeline.py`

- Added `atoms_this_source` counter per source (line 85).
- Increment counter alongside `total_atoms` for each stored atom (line 119).
- Conditional status update:
  - If `atoms_this_source > 0` → `condensed`
  - Else → `rejected` (lines 122-134)

---

## Rationale

Previously, source status was unconditionally set to `condensed` after the atom storage loop, even if zero atoms were stored. This caused false success signals downstream.

Now, the status accurately reflects extraction outcome:
- `condensed` = success (atoms produced)
- `rejected` = no atoms extracted (input produced no valid output)
- `error` = exception during processing (existing path)
