# Phase 03.0 — Canonical Authority Lock — Execution Plan

**Phase**: 03.0 (precedes Triad Enforcement)
**Type**: Implementation (hard break)
**Status**: Ready for execution
**Date**: 2026-03-27

---

## Mission

Enforce Postgres (V3) as the single source of truth. Remove all V2 runtime dependencies from the V3 codebase. Eliminate fallbacks, HybridRetriever, and dual-write paths.

---

## Success Criteria (Binary Gate)

Phase 03.0 is complete when **all** are true:

- [ ] `SystemManager.initialize` does **not** instantiate `MemoryManager`
- [ ] No import of `src.memory.manager` in any V3 component (`src/core/`, `src/research/`)
- [ ] `HybridRetriever` is removed from V3 runtime (deleted or moved outside `src/research/reasoning/`)
- [ ] All queries use `V3Retriever` only (no fallback to V2)
- [ ] System boots and accepts `/query` with V3-only storage
- [ ] No code path reads from V2 `semantic_memory` during normal V3 operation

---

## Task Breakdown

### Task 1: Remove MemoryManager Initialization

**Objective**: Eliminate `self.memory` from `SystemManager`.

**Actions**:
1. In `src/core/system.py`:
   - Delete line: `self.memory = MemoryManager()`
   - Delete line: `await self.memory.initialize()`
   - Delete line: `self.memory.set_ollama_client(self.ollama)`
   - Remove `from src.memory.manager import MemoryManager` import
2. Update any references to `self.memory` within `SystemManager` and verify none remain for V3 operation

**Acceptance**:
- `SystemManager` class compiles without `MemoryManager` import
- `SystemManager` instance has no `memory` attribute

---

### Task 2: Remove HybridRetriever

**Objective**: Prevent V3 code from using V2 retriever.

**Actions**:
1. Check if any code imports `HybridRetriever` outside of legacy paths:
   ```bash
   grep -r "from.*retriever import HybridRetriever" src/
   ```
2. If found in V3 components (e.g., `system.py`, `commands.py`), replace with `V3Retriever` or remove
3. Move `HybridRetriever` class definition to a `legacy/` directory or delete if unused
4. Ensure `src/research/reasoning/__init__.py` does not export `HybridRetriever`

**Acceptance**:
- No V3 component imports `HybridRetriever`
- `HybridRetriever` not present in `src/research/reasoning/` (or clearly marked legacy outside active path)

---

### Task 3: Enforce V3-Only Retriever Wiring

**Objective**: Confirm `SystemManager` uses only `V3Retriever`.

**Actions**:
1. In `src/core/system.py:133` (or similar), ensure line reads:
   ```python
   self.retriever = V3Retriever(adapter=self.adapter)
   ```
2. Remove any conditional logic that might switch between retriever types
3. Verify `V3Retriever` does not internally reference `self.memory` (it already uses `self.adapter`)

**Acceptance**:
- `self.retriever` is instance of `V3Retriever`
- No fallback assignment paths

---

### Task 4: Block V2 Read Paths (Defensive)

**Objective**: Add explicit guards to prevent accidental V2 usage.

**Actions**:
1. In `src/memory/manager.py`, add runtime error if instantiated in V3 mode:
   ```python
   class MemoryManager:
       def __init__(self):
           raise RuntimeError("MemoryManager is deprecated. Use SheppardStorageAdapter.")
   ```
   *Better*: Remove file entirely if no other subsystem depends on it. Check if other services (e.g., plane django app) import it. If so, move to `src/legacy/` or guard.
2. Search for any remaining `adapter.` calls that might accidentally use V2 methods — ensure adapter is V3 only
3. Add module-level assertion in V3 modules that `MemoryManager` is not imported

**Acceptance**:
- Attempting to import `MemoryManager` in V3 runtime raises clear error
- No V2 code paths reachable from `SystemManager`

---

### Task 5: Verify System Boots V3-Only

**Objective**: Confirm application starts with V3 adapter alone.

**Actions**:
1. Run a dry-start: `python3 -c "from src.core.system import SystemManager; import asyncio; sm = SystemManager(); asyncio.run(sm.initialize())"`
2. Check logs for any attempt to connect to V2 DB
3. Verify no `semantic_memory` pool created
4. Issue a simple `/query` to test retrieval path

**Acceptance**:
- Application starts without importing `MemoryManager`
- No connection to `semantic_memory` database
- Query returns results from V3 Chroma/Postgres

---

### Task 6: Documentation Update

**Objective**: Document the authority lock in code and architecture docs.

**Actions**:
1. Update `src/ARCHITECTURE.md` to state V3 is canonical, V2 is deprecated
2. Add comment header to `SystemManager` explaining V3-only invariant
3. Update `PHASE-03-VERIFICATION.md` (or create) with gate checks

**Acceptance**:
- Architecture document reflects the lock
- Code comments explain the rationale
