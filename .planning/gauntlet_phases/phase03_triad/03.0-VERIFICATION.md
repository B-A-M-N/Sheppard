### **R2 — Identity Model Locked to mission_id** (A4)

**Files**: `src/core/system.py`, `src/research/acquisition/frontier.py`

**Changes**:
- `SystemManager.learn()`: eliminated separate `topic_id`; `mission_id` is canonical

**Result**: V3 runtime exclusively uses `mission_id`. `topic_id` only appears in legacy compatibility bridges (e.g., budget.register_topic(topic_id=mission_id)) and is explicitly marked for migration.
