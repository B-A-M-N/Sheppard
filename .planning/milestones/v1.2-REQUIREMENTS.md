## Original Requirements & Outcomes

### Derived Claim Engine & Validation

- [x] **DERIV-01:** Deterministic derived claim transformations (delta, percent_change, rank) without LLM.
  - **Outcome:** ✅ Achieved. DerivationEngine implements all three original rules plus four expansions (ratio, chronology, support_rollup, conflict_rollup). All 7 rules are pure Python functions with skip-on-failure. Claim IDs are SHA-256 based for determinism.

- [x] **DERIV-02:** Extended validator to detect numeric derived claim mismatches via multi-atom recomputation.
  - **Outcome:** ✅ Achieved. `validate_response_grounding` in `src/retrieval/validator.py` now detects multi-citation numeric claims and verifies correctness by recomputing from cited atoms. `derived_mismatch` errors correctly flag incorrect LLM outputs.

### Evidence Graph & Analytical Reasoning

- [x] **GRAPH-01:** Build evidence/claim graph from EvidencePacket, including contradiction nodes.
  - **Outcome:** ✅ Achieved. `EvidenceGraph` and `ClaimGraph` implemented. Graph stores nodes (evidence, derived_claim, contradiction) with metadata and supports topological clustering. `index_by_entity` enables section planning.

- [x] **GRAPH-02:** Analytical operators (trend, difference, variance, correlation, significance, outlier) as pure functions.
  - **Outcome:** ✅ Achieved. All six operators implemented in `src/research/reasoning/analytical_operators.py` as deterministic pure functions. Comprehensive tests cover edge cases and numerical stability.

### Section Planning

- [x] **PLAN-01:** Evidence-aware section planner that converts EvidenceGraph into EnrichedSectionPlans.
  - **Outcome:** ✅ Achieved. `EvidenceAwareSectionPlanner` consumes graphs and produces `EnrichedSectionPlan` objects with modes (DESCRIPTIVE, COMPARATIVE, ADJUDICATIVE, IMPLEMENTATION, SURVEY), evidence budgets, length ranges, contradiction obligations, and allowed derived claim scopes.

### Multi-Pass Composition

- [x] **SYNTH-01:** Multi-pass composition pipeline (5 passes) that transforms plans + packets into ReportDraft.
  - **Outcome:** ✅ Achieved. `MultiPassSynthesisService` in `synthesis_service_v2.py` implements all 5 passes: Pass 1 (first draft), Pass 2 (expansion conditional on threshold), Pass 3 (transition coherence), Pass 4 (grounding repair), Pass 5 (placeholder for verifier). Output types `SectionDraft` and `ReportDraft` defined.

### Longform Verification

- [x] **VERIFY-01:** Longform verifier with 7 gates (sentence grounding, derived recomputation, contradiction obligation, evidence threshold, no uncited abstraction, expansion budget, quality metrics).
  - **Outcome:** ✅ Achieved. `LongformVerifier` implemented with deterministic gates. HARD gates (1-5) cause `is_valid=False` on failure; SOFT gate (6) emits warnings; METRICS gate (7) produces quality metrics. 13 tests cover all gate behaviors and failure harness.
