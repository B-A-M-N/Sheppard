# Phase 12-02.1: Retrieval Latency Diagnosis Summary

A small investigative sub-phase created after verification of 12-02 showed PERF-01 not met (retrieval ~1066ms vs ≤200–300ms target). We instrumented the retrieval pipeline to profile query times and ran a test matrix.

Key findings:
- Single-query retrieval: 144–184ms (good)
- 8-concurrent queries: per-section ~1s, total ~1.3s (bad)
- Efficiency ~6.7× indicates near-serialization
- Root cause: embedding computation runs in thread pool, but GIL prevents parallel CPU execution → contention.
- Corpus size scaling: sublinear; bloat not a factor.

Recommendation: Batch section queries into a single Chroma call to avoid per-query embedding overhead and GIL contention (see ANALYSIS.md for details).

All artifacts committed; phase complete.
