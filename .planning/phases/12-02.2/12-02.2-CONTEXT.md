# Context

Phase 12-02 implemented concurrent retrieval via `asyncio.gather`, but verifier found total retrieval ~1066ms, missing PERF-01 (≤200–300ms). Phase 12-02.1 diagnosed the root cause:

- Single-query latency: ~150–180ms (good)
- Concurrent (8 sections): per-section ballooned to ~1s, total ~1.3s
- Efficiency ~6.7× indicates near-serialization
- Root cause: `ChromaSemanticStoreImpl.query` runs `coll.query` in `asyncio.to_thread`. Embedding computation is CPU-bound and holds Python's GIL, so threads cannot execute in parallel. 8 concurrent tasks time-slice, inflating each duration ~7×.

Corpus size and total DB bloat are not significant factors.

The recommended fix: **batch section queries into a single Chroma call** using `query_texts=[...]`. This computes embeddings once for all queries and performs vector search in one round-trip, eliminating per-query contention.

# Goal

Achieve PERF-01: total retrieval latency (8 sections) ≤200–300ms by implementing batched retrieval while preserving all truth invariants and determinism.

# Scope

- Modify `V3Retriever` to support batched retrieval: new method `retrieve_many(List[RetrievalQuery]) -> List[RoleBasedContext]`
- Modify `EvidenceAssembler.assemble_all_sections` to optionally use batched retrieval when `len(sections) > 1` (or always use batched path)
- Ensure deterministic results: same atom_ids_used ordering, same citation keys, same per-section content as the sequential baseline.
- Re-run benchmark suite (Wave 3 script) to measure improvement against PERF-01 across small/medium/large tiers.
- Run full guardrail tests to confirm no regressions.

# Non-Negotiable

- mission_id filtering must work for each section's context (include `where` per section; Chroma allows `where` per query text in batch? Need to check API. Likely we need to pass same `where` for all queries if mission_id same; that's fine.)
- retrieval_profile (if needed) must be preserved per-section for possible diagnostics.
- All truth invariants from Phase 10/11/12 must remain satisfied.

# Success Criteria

- Benchmark on small corpus (20 atoms, 8 sections) shows total retrieval_ms ≤ 300ms.
- Batched result sets are functionally equivalent (modulo order) to sequential baseline.
- All guardrail tests pass.
- No regression in deterministic ordering.
